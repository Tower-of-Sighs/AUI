# ApricityUI `container` / `slot` / `recipe` Guide

Last updated: 2026-02-18

## 1. Overall Model

The current unified semantics are:

- Only one slot tag is used: `<slot>`
- `slot` is split by `mode`:
  - `bound`: binds to a real menu slot
  - `virtual`: display-only slot
- `container` uses `layout` to describe slot layout only
- Container titles are read only from the text of the first child element, such as `div` or `span`
- Slots generated by `recipe` are always `virtual`

## 2. `container` Attributes

### 2.1 `bind`

- Binds a data source such as `player`, `saved_data`, `block_entity`, or `entity`

### 2.2 `layout`

- Slot layout rules, either grid-style or presets
- Examples:
  - `layout="[27,3,9]"`
  - `layout="preset:player"`

Note: `layout` only affects arrangement. In normal cases it does not automatically create missing slot instances.

### 2.3 Title Rules

- The title is rendered inside the container
- The title source rule:
  - Only the text of the first child element is read
  - If the first child is missing or empty, the title area is not rendered and no placeholder is kept

Example:

```html
<container primary="true" bind="player" layout="preset:player">
  <div class="demo-title">My Title</div>
  <slot mode="bound" slot-index="0" repeat="36"></slot>
</container>
```

## 3. `slot` Attributes

### 3.1 `mode`

- `bound|virtual`
- Default behavior when not declared explicitly:
  - Inside `container`: `bound`
  - Outside `container`: `virtual`

### 3.2 `repeat`

- Batch generation using total-count semantics
- `repeat="36"` means a total of 36 slots, counting the template node itself as index 0

### 3.3 Common `bound` Attributes

- `slot-index`: local slot index
- `disabled`: disables interaction
- CSS such as `position/top/left/...` can still be used for manual layout

### 3.4 Common `virtual` Attributes

- `innerText`: item literal, and this is the only supported entry
  - Item id: `minecraft:diamond`
  - Tag: `#minecraft:planks`
  - Id plus NBT: `minecraft:diamond_sword{Damage:12}`
  - Full ItemStack NBT: `{id:"minecraft:diamond",Count:1b,tag:{display:{Name:'{"text":"Preview"}'}}}`
- `rotate-interval`: candidate-rotation interval for tag-based items, in ms

## 4. Player Container Default 36 Slots

When `container.bind="player"` and there are no bound slots inside the container, the system injects player inventory slots implicitly:

- `inv`: 27 slots (3x9)
- `hotbar`: 9 slots (1x9)
- Default gap: `4px`

If you already declare bound slots explicitly, this implicit injection does not happen.

## 5. `recipe` Rules

- `<recipe>` can be used independently in normal HTML display
- Multiple `<recipe>` tags may exist inside a container, each with its own sub-layout
- Slots generated by `recipe` are always `virtual` and never bind to a real menu
- Syntax: `<recipe type=\"...\">recipe_id</recipe>`
- `type` is required and validated strictly. Recommended values:
  - `crafting_shaped`
  - `crafting_shapeless`
  - `smelting` / `blasting` / `smoking` / `campfire_cooking`
  - `stonecutting`
  - `smithing`

## 6. Examples And Entry Points

- `minecraft:diamond` -> `run/apricity/test/index.html`
- `minecraft:emerald` -> `run/apricity/test/saveddata_player.html`
- `minecraft:amethyst_shard` -> `run/apricity/test/virtual_container.html`
- `minecraft:nether_star` -> `run/apricity/test/recipe_showcase.html`

Example snippet:

```html
<container primary="true" bind="saved_data" layout="[9,3,3]">
  <div class="demo-title">SavedData Storage (9 slots, primary)</div>
  <slot mode="bound" repeat="9" slot-index="0"></slot>
</container>

<container bind="player" layout="preset:player">
  <div class="demo-title">Player Inventory (36 slots)</div>
  <slot mode="bound" repeat="36" slot-index="0"></slot>
</container>

<slot mode="virtual">{id:"minecraft:diamond",Count:12b,tag:{display:{Name:'{"text":"Example Item"}'}}}</slot>
<recipe type="crafting_shaped">minecraft:crafting_table</recipe>
```
